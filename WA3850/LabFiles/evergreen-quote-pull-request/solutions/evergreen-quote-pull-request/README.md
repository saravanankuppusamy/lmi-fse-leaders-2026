# Evergreen Quote — Pull Request (solution)

This is the Week 6 solution. The page code itself is unchanged from
the Week 5 starter *except* for one string in `index.html`:

```diff
    <section class="hero">
-     <h1>Coverage that grows with you</h1>
+     <h1>Coverage that scales with your life</h1>
      <p class="tagline">Get a personalized quote in under a minute.</p>
    </section>
```

That one-line change is what the student edited on the
`update-hero-copy` feature branch, committed, pushed to their fresh
GitHub repository (`evergreen-quote-demo`), opened as a pull request,
self-reviewed, and squash-merged into `main`.

The real output of this week's lab is not the string change — it is
the *workflow*. A leader who has opened, reviewed, and merged a real
GitHub PR once will read every future PR with more confidence.

See `evergreen-quote-pull-request.adoc` in the course materials for
the full walkthrough.
